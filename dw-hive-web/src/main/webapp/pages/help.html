查询步骤指引

查询步骤如下：
  1. 选择对应的数据库,执行引擎,用户权限角色(如果没有对应表的读权限, 请申请)；如果查询表的数据量不大的情况下，请尽量使用shark,查询速度相对较快！
  2. 在查询tab页里输入需要执行的sql语句，对于多条语句使用分号隔开； 如果查询语句太多，可以打开多个tab页分别输入语句进行查询。
  3. 语句输入完毕后，选择需要查询获取的记录数目，点击执行(快捷键：ctrl+enter)。
  4. 点击执行后，在下方会立马出现sql查询的进度信息，信息会不断刷新。待执行结束后，显示结果的tab页会立马显示出来；在结果tab 页里，显示了部分的结果数据；如果觉过数据量很大，请下载csv文件或者excel文件进行查看！

查看查询历史记录

查询步骤如下：
  1. 点击导航栏的 历史查询 ，会显示登录用户的所有的历史查询sql
  2. 在表格的右上角搜索框里 可以通过关键字 快速查找出所需的sql语句
  3. 每条查询语句都保存了csv和excel格式的结果文件，可以下载查看

创建临时表&上传数据

创建临时表步骤如下：
  1. 在最上方选择建表的用户身份
  2. 填写需要创建的临时表相关信息：所在库，表名，列分隔符，文件存储格式，字段等等；星号标记的位置必须填写！
  3. 信息输入完成后，点击生成建表语句，获取建表语句，检查语句的正确性
  4. 语句确认无误后，点击执行建表
临时表上传数据步骤如下：
  1. 输入表信息，如果选择覆盖，即本次上传的数据将覆盖掉该表的历史数据
  2. 选择数据文件，可以多选(注意文件的格式和字段分隔符和临时表信息一一对应)
  3. 文件选择完毕后，点击upload，将文件上传到hdfs上(如果不需要上传了，可惜点击remove删除，cancel是取消上传过程)
  4. 最后点击提交，完成数据Load到临时表的过程

常见问题

  1. 如果发现点击执行后，日志显示一段时间后，日志中断刷新？

    2. 出现这个问题的可能原因是 网络不好，导致定时发送请求到后端获取日志的过程停止；
    解决办法： 点击 重试(因网络原因) 按钮

  2. 如果发现提交的sql语句不需要在执行？

    3. 直接点击stop停止查询，修改语句后 再次提交
